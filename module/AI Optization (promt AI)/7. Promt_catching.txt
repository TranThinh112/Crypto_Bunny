Hãy thiết kế lại toàn bộ Prompt Engine của hệ thống AI Trading để tối ưu Prompt Caching của OpenAI.

Mục tiêu:

- Tận dụng Prompt Caching tối đa.
- Giảm chi phí API.
- Giảm latency.
- Dễ bảo trì.
- Dễ mở rộng.
- Không làm thay đổi logic trading hiện tại.
- Hỗ trợ GPT-5.4-mini và GPT-5.5.
- Hỗ trợ Prompt Versioning.
- Hỗ trợ Model Versioning.
- Hỗ trợ AI Decision Versioning.
- Hỗ trợ A/B Testing Prompt.

Tech stack

- ASP.NET Core Web API
- C#
- MongoDB

======================================================
MỤC TIÊU

Đảm bảo:

- GPT-5.4-mini và GPT-5.5 dùng chung Prompt Engine.

- Chỉ Market JSON thay đổi giữa các request.

- Tận dụng Prompt Caching tối đa.

- Có thể rollback Prompt Version bất kỳ.

- Có thể chạy A/B Testing Prompt.

- Theo dõi Prompt Version nào cho:
  + Winrate cao nhất
  + Profit Factor cao nhất
  + Drawdown thấp nhất
  + RR tốt nhất
  + Latency thấp nhất
  + Chi phí API thấp nhất

- Sau mỗi TradeExecution phải lưu đầy đủ:
  + PromptVersion
  + PromptHash
  + ModelName
  + ModelVersion
  + ExperimentName

để có thể thống kê chính xác hiệu quả của từng Prompt Version trong tương lai.
