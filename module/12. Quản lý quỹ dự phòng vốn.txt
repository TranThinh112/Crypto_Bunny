PROMPT 2 — MODULE CAPITAL RESERVE MANAGER
Tên tiếng Việt: Quản lý quỹ dự phòng vốn

Hãy tạo module Capital Reserve Manager cho bot AI trading crypto.

Mục tiêu:

* Quản lý quỹ dự phòng vốn dựa trên realized_capital.
* Tách vốn thành:

  * RealizedCapital
  * ReserveCapital
  * TradingCapital
* Không cho Trading Engine sử dụng toàn bộ vốn tài khoản.
* Không dùng PnL thả nổi để tính trading_capital.
* Không dùng equity để tính trading_capital.
* Không tự mở lệnh.
* Không sửa rule trading.
* Không gọi GPT.
* Module chỉ kiểm tra, tính toán và trả quyết định cho Trading Engine.

Tech stack:

* Python 3.11+
* MongoDB Atlas
* SQLAlchemy
* FastAPI
* Pydantic

Cấu hình:

Database MongoDB Atlas:

Tạo bảng capital_reserve_states:

* id
* created_at
* updated_at
* mode: HEALTHY / WARNING / RECOVERY / CRITICAL
* realized_capital
* reserve_percent
* reserve_amount
* trading_capital
* used_trading_capital
* available_trading_capital
* allow_reserve_usage
* reason

Module này phải đọc realized_capital mới nhất từ bảng capital_snapshots do Capital Sync Service tạo ra.

Tạo Options:

* CapitalReserveOptions

Capital Reserve Logic:

Tạo CapitalReserveService với các hàm:

* calculate_state(mode, used_trading_capital)
* get_current_state()
* refresh_state(mode, used_trading_capital)
* get_available_trading_capital()
* can_allocate_margin(required_margin)
* reserve_usage_allowed()

Chọn reserve_percent theo mode:

* HEALTHY → BaseReservePercent
* WARNING → WarningReservePercent
* RECOVERY → RecoveryReservePercent
* CRITICAL → CriticalReservePercent

Công thức bắt buộc:

* reserve_amount = realized_capital * reserve_percent / 100
* trading_capital = realized_capital - reserve_amount
* available_trading_capital = trading_capital - used_trading_capital

Logic bắt buộc:

* Nếu available_trading_capital < required_margin thì reject mở lệnh.
* Nếu trading_capital < MinTradingCapital thì reject hoặc pause trading.
* Nếu AllowReserveUsage = false thì không được dùng reserve_amount để mở lệnh mới.
* Nếu mode = CRITICAL và EmergencyAllowReserveUsage = false thì reject mọi lệnh mới.
* Nếu mode = CRITICAL và EmergencyAllowReserveUsage = true thì chỉ cho phép dùng reserve để duy trì margin, không dùng để mở lệnh mới.
* Capital Reserve Manager chỉ được dùng realized_capital.
* Không được dùng equity.
* Không được dùng wallet_balance + unrealized_pnl.
* Không được dùng unrealized_pnl để tăng khối lượng lệnh.
* Không được mở lệnh thay Trading Engine.
* Chỉ trả kết quả allowed / rejected và reason.

Tích hợp hệ thống:

Mode được xác định từ các module khác:

1. Bunny Health Monitor WARNING → mode = WARNING.
2. Bunny Minimize Losses Recovery Mode → mode = RECOVERY.
3. Recovery Chain Manager active chains vượt ngưỡng → mode = RECOVERY.
4. Bunny Health Monitor CRITICAL → mode = CRITICAL.
5. Nếu không có cảnh báo → mode = HEALTHY.

Validate Entry:

Khi Trading Engine chuẩn bị mở lệnh:

* Lấy required_margin.
* Gọi CapitalReserveService.can_allocate_margin(required_margin).

Nếu false:

* Reject với reason:
  "Insufficient trading capital after reserve protection"

API FastAPI:

1. GET /capital-reserve/state

Trả về:

* mode
* realized_capital
* reserve_percent
* reserve_amount
* trading_capital
* used_trading_capital
* available_trading_capital
* allow_reserve_usage
* reason

2. POST /capital-reserve/refresh

Input:

* mode
* used_trading_capital

Chức năng:

* Lấy realized_capital mới nhất từ capital_snapshots.
* Tính reserve_amount.
* Tính trading_capital.
* Tính available_trading_capital.
* Lưu state mới vào capital_reserve_states.
* Phải dùng transaction khi refresh state.

3. POST /capital-reserve/check-allocation

Input:

* required_margin

Output:

* allowed: true/false
* reason
* available_trading_capital
* reserve_amount
* realized_capital
